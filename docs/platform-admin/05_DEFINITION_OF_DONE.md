## Definition of Done (LOCKED)

A phase is considered complete only when all of the following are true:

1. All approved requirements have been implemented.

2. Acceptance criteria have been fully satisfied.

3. No critical or blocking defects remain.

4. Existing functionality has been regression tested.

5. Documentation has been updated.

6. User interface has been reviewed for consistency.

7. Database changes (if any) have been verified.

8. Security and permissions have been validated.

9. Build passes successfully.

10. Changes have been committed with a clear commit message.

11. The phase has been reviewed, approved, and officially locked before the next phase begins.
